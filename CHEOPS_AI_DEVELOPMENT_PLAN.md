# Cheops AI: План Развития Продукта и Платформы

Дата: 21 марта 2026  
Статус: Draft v1 (рабочий план для реализации)

## 1) Контекст и Цель

Проект меняет позиционирование и название: **APRIS -> Cheops AI**.  
Цель плана: превратить текущий MVP в устойчивую, масштабируемую аналитическую платформу для раннего выявления пирамидальных рисков, с фокусом на:

- редизайн пользовательского опыта и интерфейса;
- усиление ML-составляющей;
- надежную инженерную и MLOps-базу для эксплуатационного режима.

## 2) Целевое Видение Cheops AI

Cheops AI — это Risk Intelligence Platform, где:

- аналитик получает стабильный поток алертов с прозрачным приоритизатором;
- руководитель видит качество модели, объемы обработки и риски по сегментам;
- инженерная команда может безопасно выпускать улучшения без деградации модели и UX.

## 3) Ключевые Направления Развития

## 3.1 Редизайн и UX

- Полный редизайн визуальной системы под бренд Cheops AI.
- Пересборка пользовательских сценариев: `Сканирование -> Очередь алертов -> Досье -> Решение`.
- Введение дизайн-системы: типографика, цветовые токены, компоненты, состояния, паттерны данных.
- Усиление explainability в интерфейсе: понятные причины риска, доверие к скору, контекстные подсказки.

## 3.2 ML и Данные

- Переход к формальной ML-постановке с бизнес-метриками.
- Усиление ETL для реальных источников данных (CSV/JSON + унифицированные схемы).
- Бенчмаркинг и выбор лучшего табличного алгоритма (LightGBM/CatBoost/XGBoost).
- Калибровка вероятностей, мониторинг дрифта, план переобучения.

## 3.3 Архитектура и Эксплуатация

- Закрепление API-first архитектуры (UI только через HTTP API).
- Batch scoring через отдельный endpoint и асинхронную обработку.
- Контейнеризация (Docker + docker-compose), предсказуемые окружения.
- Наблюдаемость: health, логи, метрики, контроль деградаций.

## 3.4 Качество и Процессы

- Полный тестовый контур: unit + api + smoke + контрактные тесты.
- Quality gate: pytest, ruff, mypy, pre-commit, CI.
- Регламент релизов и откатов (модель, API, UI).

## 4) KPI и Критерии Успеха

Минимальные целевые KPI на этап Cheops AI v1:

- ML:
- ROC-AUC >= 0.97 на контрольном датасете.
- Recall класса высокого риска >= 0.92 при контролируемом FP-rate.
- Калибровка вероятностей: Brier score улучшается относительно текущего baseline.

- UX/Product:
- Время до ключевого действия аналитика (от открытия до решения по алерту) сокращено на 30%.
- Не менее 80% тестовых сценариев выполняются без ошибок UX.
- Доля успешных batch-обработок >= 99%.

- Reliability:
- Uptime локального стенда >= 99% в рабочих окнах.
- P95 ответа API `/predict` <= 300ms (без batch).
- Все релизы проходят CI quality gate.

## 6) План Редизайна (Детализация)

Цели редизайна:

- сделать интерфейс меньше “демо” и больше “операционным”;
- ускорить принятие решений аналитиком;
- повысить доверие к объяснениям модели.

Обязательные изменения:

- Единая шапка продукта: **Cheops AI**.
- Консистентная навигация между модулями.
- Табличный alert inbox с фильтрами и приоритизацией.
- Досье как центральный рабочий экран:
- summary риска;
- сетевой контекст;
- вклад факторов;
- история изменений.
- Паттерны ошибок:
- явные пустые состояния;
- понятные сообщения при недоступности API;
- быстрые действия восстановления.

Критерии приемки редизайна:

- ключевые сценарии проходят без навигационных тупиков;
- время до действия сокращается относительно baseline;
- визуальная консистентность подтверждена чек-листом дизайн-системы.

## 7) План Улучшения ML (Детализация)

ML-цели:

- повысить детекцию высокорисковых кейсов;
- сделать вероятности калиброванными;
- контролировать качество на новых данных.

Обязательные ML-инициативы:

- Data quality checks в ETL:
- схема полей;
- диапазоны;
- доля пропусков;
- аномальные распределения.
- Feature governance:
- документ признаков;
- стабильность распределений;
- версия трансформаций.
- Model governance:
- версия модели;
- версия датасета;
- протокол сравнения кандидатов.
- Мониторинг:
- drift по признакам;
- drift по предсказаниям;
- метрики качества на контрольных выборках.

Критерии приемки ML-улучшений:

- фиксируемый прирост ключевых метрик против baseline;
- стабильность объяснений на контрольном наборе кейсов;
- отсутствие деградаций API SLA.

## 9) Риски и Снижение Рисков

- Риск: нестабильность локального запуска (процессы завершаются).  
Митигирующие действия:
- фоновые процессы с логами;
- проверки health;
- единая команда запуска и остановки;
- диагностика через `.run/*.err.log`.

- Риск: переусложнение UX без роста пользы.  
Митигирующие действия:
- измеримые UX KPI;
- сценарное тестирование до релиза;
- итеративный rollout.

- Риск: ML-улучшения не дают прироста на реальных кейсах.  
Митигирующие действия:
- строгий benchmark протокол;
- holdout-наборы;
- контрольная эксплуатация перед full rollout.

## 10) Definition of Done для Cheops AI v1

- Брендинг и продуктовая терминология полностью обновлены на Cheops AI.
- Редизайн внедрен и подтвержден UX-критериями.
- ML v2 проходит target-метрики и calibration-check.
- Платформа запускается воспроизводимо и стабильно.
- CI quality-gate обязателен для каждого merge.
- Документация актуальна и покрывает запуск, тестирование, релиз, откат.
